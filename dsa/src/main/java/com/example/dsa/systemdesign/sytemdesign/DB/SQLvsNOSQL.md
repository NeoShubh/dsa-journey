## SQL vs NoSQL Databases

| SQL Databases | NoSQL Databases |
|---------------|-----------------|
| Uses a relational data model with tables (rows and columns). | Uses non-relational models such as document, key-value, column-family, or graph. |
| Requires a fixed and predefined schema. | Follows a schema-less or flexible schema approach. |
| Generally scales **vertically**. | Designed to scale **horizontally**. |
| Follows **ACID** properties for strong consistency. | Often follows the **BASE** model focusing on availability and eventual consistency. |
| Best suited for structured and transactional data. | Best suited for unstructured data and large-scale applications. |
