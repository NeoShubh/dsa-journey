## Functional Requirements

It defines the specific features and operations a system must perform to meet the users' and stakeholders' needs.

Examples:
- User login
- Order catalog
- Payment service
- etc.

---

## Non-Functional Requirements

It defines how a system should perform, mainly focusing on performance, security, and reliability.

It focuses on how my system works in different environments, whether it is able to handle load, and how gaming platforms stream games from different locations.

**Example:** Scalability.
